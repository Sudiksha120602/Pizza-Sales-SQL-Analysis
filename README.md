# 🍕 Pizza Sales Analysis | SQL

## 📌 Project Overview

This project analyzes pizza sales data using SQL to uncover customer purchasing patterns, sales performance, product popularity, and revenue trends. By querying multiple relational tables, the analysis provides business insights that can support inventory planning, menu optimization, and sales reporting.

The project demonstrates practical SQL skills commonly used by Data Analysts, including joins, aggregate functions, subqueries, grouping, ranking, and window functions.

---

## 📂 Dataset

The analysis was performed using four relational datasets:

* **orders.csv** – Order IDs, dates, and times
* **order_details.csv** – Individual pizza items ordered and quantities
* **pizzas.csv** – Pizza sizes and prices
* **pizza_types.csv** – Pizza names and categories

The tables were linked using primary and foreign keys to create a complete sales dataset.

## Business Questions Answered

### Sales Performance

* Total number of orders
* Total revenue generated
* Highest-priced pizza
* Most common pizza size ordered

### Customer Ordering Behavior

* Hourly order distribution
* Average pizzas ordered per day
* Most ordered pizza types
* Quantity sold by pizza category

### Revenue Analysis

* Top revenue-generating pizzas
* Revenue contribution by pizza category
* Daily cumulative revenue trend
* Top-performing pizzas within each category

---

## Key Business Insights

* Calculated total sales revenue directly from transactional order data.
* Identified the highest revenue-generating pizza products.
* Determined customer ordering patterns across different hours of the day.
* Measured revenue contribution by pizza category to support product mix analysis.
* Used cumulative revenue trends to evaluate business performance over time.
* Ranked top-performing pizzas within each category using SQL window functions.
